Esri Feature Layer
ArcGIS Online or Enterprise
Esri Feature Layer The source of truth. Can be hosted on ArcGIS Online or your own ArcGIS Enterprise portal. Supports public layers (no auth) and private layers (username/password token auth). 2FA-protected accounts are not compatible — use a service account.

Records are fetched via the Feature Layer REST API with pagination (up to 1000 records per page).
REST API Poll The poller sends an HTTP GET to the Feature Layer's /query endpoint every N seconds (configurable). It requests all fields, WGS84 coordinates, and up to 1000 records per page. When delta tracking is enabled, only records modified since the last run are fetched.
📄
config.json
TAK host, auth mode, field mappings
config.json Stored at /opt/Esri-TAKServer-Sync/config.json. Defines the TAK Server host/port, auth mode (cert or plain TCP), Feature Layer URL, field mapping (lat/lon/callsign/cot_type), delta tracking settings, and icon mapping rules. Written by infra-TAK on each deploy or config save.
💾
delta-state.json
Tracks last-seen EditDate per UID
Delta State Persisted to /opt/Esri-TAKServer-Sync/delta-state.json. When delta mode is on, only records with an EditDate newer than the stored value are sent — reducing bandwidth and TAK Server load. Delete this file to force a full re-broadcast on next start.
🐍
feature-layer-to-cot.py
Python poller + CoT builder
Python Worker Script Runs as a systemd service (feature-layer-to-cot.service). On each poll cycle it:
1. Fetches all (or changed) records from the Feature Layer
2. Maps lat/lon/callsign/type fields to CoT attributes
3. Optionally resolves an icon path from the icon mapping table
4. Builds a CoT XML <event> element for each record
5. Streams them to TAK Server over the open TCP/TLS connection
6. Sends a keepalive ping every 15 s to keep the connection alive
🎨
icons/*.zip
Uploaded ATAK/WinTAK iconsets
Icon Mapping Iconset .zip files are stored in /opt/Esri-TAKServer-Sync/icons/. Each zip must contain an iconset.xml with a uid attribute. The Icons tab lets you assign a column value → iconset path mapping. The poller injects <usericon iconsetpath="uuid/group/icon.png"/> into each CoT event so ATAK/WinTAK renders the correct icon.
TLS/Cert Mode (port 8089) Uses mTLS — the client presents a .p12 certificate. TAK Server validates it against its trusted CA list. Use cd /opt/tak/certs && ./makeCert.sh Client esri-push to generate a CA-signed cert, then enroll it via UserManager.jar certmod -A. Extract PEM sidecars with openssl pkcs12 -legacy ....
cert (8089)
Plain TCP Mode (port 8087) Unencrypted TCP connection — simpler to set up but traffic is not encrypted. Only use on a private/trusted network. TAK Server must have a plain TCP input configured in CoreConfig.xml (many default TAK Server installs have only TLS enabled on 8089).

TAK Server
Streams CoT to connected clients
TAK Server Receives the CoT XML stream and fans it out to all connected clients in the configured group.

The esri-push user/cert is enrolled in a TAK group (e.g. __ANON__ or a custom group). All clients subscribed to that group will see the PLIs.

Stale time is configurable — records disappear from maps after N minutes if not refreshed.
CoT Distribution TAK Server sends CoT events to all connected ATAK, WinTAK, iTAK, and TAKX clients. Each Feature Layer record appears as a PLI (Position Location Information) dot on the map with its callsign label and — if icon mapping is configured — a custom icon.
📱
ATAK
Android TAK
ATAK (Android) Feature Layer records appear as PLI markers. Custom icons render if the matching iconset UUID is installed on the device. Remarks fields appear in the detail callout.
💻
WinTAK
Windows TAK
WinTAK (Windows) Same CoT stream. The bundled Incident Icons iconset is already installed in WinTAK by default — icons from that set will render immediately.
🌐
TAK Web / iTAK
Browser / iOS
TAK Web / iTAK Any CoT-compatible client connected to the same TAK Server and group will receive the Feature Layer records in real time.
